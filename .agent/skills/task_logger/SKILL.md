# 任務紀錄自動維護 Skill

## 觸發條件

**每一次**使用者發送 prompt 並且你完成工作後，在最終回覆使用者**之前**，你**必須**執行以下步驟來維護任務紀錄。

## 執行步驟

### 1. 讀取或建立紀錄檔

- 檢查專案根目錄 `d:\IOT_HW1\任務紀錄.md` 是否存在。
- 若**不存在**，建立新檔案，開頭為：

```markdown
# 📋 IoT 專題 — 任務紀錄

> 此文件由 AI 自動維護，記錄每次對話的指令與執行細節。越上方的紀錄越新。
```

### 2. 插入新紀錄

在檔案標題區塊（`# 📋 IoT 專題 — 任務紀錄` 及其下方的說明引言）**之後**、所有既有紀錄**之前**，插入一個新的紀錄區塊。

### 3. 紀錄格式

每筆紀錄使用以下格式：

```markdown
---

## 🕐 YYYY-MM-DD HH:MM (時區，例如 +08:00)

### 📝 使用者指令

> [完整引用使用者的原始 prompt 內容]

### 🔧 執行內容

- [具體描述做了什麼事]
- [修改或新增了哪些檔案，附上路徑]
- [使用了哪些工具或指令]
- [遇到的問題與解決方式（如有）]
- [其他重要細節]
```

### 4. 紀錄撰寫原則

- **時間**：使用系統提供的當前時間，格式為 `YYYY-MM-DD HH:MM (+08:00)`。
- **使用者指令**：完整引用使用者的 prompt，不要省略或摘要。
- **執行內容**：詳細列出所有操作步驟，不僅是大綱。包含：
  - 建立、修改、刪除了哪些檔案（附路徑）
  - 執行了哪些終端指令
  - 安裝了哪些套件
  - 遇到了什麼錯誤以及如何解決
  - 設計決策的理由
- **排序**：越新的紀錄在越上面（插入在標題區塊之後、舊紀錄之前）。

### 5. 使用工具

- 使用 `view_file` 工具讀取現有的 `任務紀錄.md`。
- 使用 `replace_file_content` 或 `write_to_file`（搭配 `Overwrite: true`）來更新檔案。
- 插入時要保持既有紀錄不變，只在頂部（標題之後）加入新紀錄。

## 重要提醒

> [!CAUTION]
> 此 Skill 必須在**每次對話**中執行，無論使用者的 prompt 內容為何。即使只是回答問題，也要記錄下來。絕對不可遺漏。

> [!IMPORTANT]
> 紀錄內容要**詳細且具體**，不要只寫大綱式的摘要。使用者希望能從紀錄中完整回溯每次操作的所有細節。
